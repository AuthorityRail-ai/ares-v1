# Contributing to ARES-v1

ARES-v1 is an open standard published under the AuthorityRail Standards Foundation. Contributions are welcome.

## How to contribute

Open an issue to propose a change to the specification. Describe the problem the change solves and the proposed amendment. Changes to published versions (v1.0.0) are not accepted — the canonical spec is immutable. Proposed changes target the next version.

Submit a pull request with your proposed specification change. All pull requests must include a rationale section explaining why the change strengthens the standard.

Join the discussion at hello@authorityrail.com or the AuthorityRail Discord.

## What can be contributed

New evaluation layer definitions. Additional compliance framework mappings. Implementation guides for new languages or frameworks. CAR schema extensions proposed for v1.1. Corrections to documentation that do not alter the specification.

## What cannot be changed

The three core decisions: ALLOW, DENY, ESCALATE. The requirement that every decision produces a signed CAR. The principle that authorization must occur before execution. The immutability of published CAR records. These are the laws. They do not change.

## Code of conduct

All contributions must stay in the infrastructure register. No marketing language. No product comparisons. No hype. The standard speaks in the language of law and enforcement.

Every autonomous action requires authority before execution.
